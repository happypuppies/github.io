Phase 1: Data Acquisition and Preprocessing

Challenges Encountered

  • High resolution elevation data is expensive or incomplete.
  • High resolution aerial imagery data for model verificatin was also hard to find.

Outcomes

  • Ontario GeoHub provided great elevation data in their Ontario Integrated Hydrology dataset, as well as cold water data in their Aquatic Resource Area data.
  • Data was extracted by mask to the Area Of Interest and Projected to NAD 83 Ontario MNR Lambert.

Recommendations

  • Higher resolution elevation data would yield more reliable results. NRCAN Geogratis data has a resource with 1 metre elevation data but it is not currently complete over the Area Of Interest but this data set could be revisited at a later date to update the model created in this study.

Phase 2: Spatial Analysis

Challenges Encountered

  • Though we had hoped to include our Trace Downstream and Trace Upstream tools in the delivered Experience it was found that the NCC AGOL account does not allow the use of trace networks, and no way was found to replicate this functionality with the tools available, so it was left as desktop only.
  • Experience Builder was a steep learning curve with a tricky to use interface (ie. feature view widget overrides initial map view settings)
  • StoryMaps didn't provide enough functionality and is optimized for multi-media and static content, which was not the focus of the solution

Outcomes

  • Experience Builder contains a lot of useful functionality but requires a significant amount of time to learn
  • Sometimes using a "workaround" is easier than trying to get the widget to work in the indended manner (such as manipulating the original data instead of recoding a custom widget)
  • The choice of the solution is directly dependent on the goal of the project

Next Steps

  • Dedicated more time to learning Exprience Builder

Milestone 3.1: Development in ArcGIS Experience Builder

Challenges Encountered

  • Showing stream interconnectivity was difficult to do using the configurable widgets
  • Group editing was not possible on Experience Builder (group members could not be editing the same Web Experience at once)
  • Dynamic text is difficult to link to the appropriate data source
  • Using views was difficult, but were required to overcome challenges of widgets re-setting initial view
  • Difficulty in distinguishing format options between Maps, Views, and Pages

Learning Outcomes

  • Created multiple prototype proof-of-concept Experiences utilizing different widgets to display different properties of the data
  • Possible to create a webpage-style application in Experience Builder

Next Steps

  • Decide which approaches best fit the goals of the problem statement

Milestone 3.2: Download Developer Edition

Challenges Encountered

  • Downloading Developer Edition was time consuming and required the download of Node.js, ArcGIS Experience Builder Developer Package (requires running multiple programs from the Command Prompt)
  • Every usage requires starting the client and server services from Command Prompt
  • To authenticate the application, new credentials had to be created in a ArcGIS Developers account
  • To make custom widgets it is necessary to learn TypeScript and React which are ESRI specific languages

Learning Outcomes

  • Getting ArcGIS Experience Builder Developer Edition started on a local computer is time-consuming (valuable time taken away from working on the actual project)
  • The learning curve for create custom widgets is extremely high and should be avoided if possible

Next Steps

  • Return to Experience Builder to focus on configuring the "off-the-shelf" widgets

Milestone 4.1: Progress Review with Shawn

Meeting Outcomes

  • Recommended using ArcGIS Hub to create a website to embed the Experience Builder solutions and create a webpage-style application
  • Encouraged the manipulation of data instead of configuring the widget

Milestone 4.2: Introduction to ArcGIS Hub

Challenges Encountered

  • Default template contained a lot of features that were not necessary (subtractive rather than additive templates)
  • Challenging to change page formatting
  • Easy to lose changes when switching to Preview Mode
  • Difficult to find ArcGIS Hub in AGOL account as it creates a new folder upon creation

Learning Outcomes

  • ArcGIS Hub is a viable option for combining static and dynamic ESRI content (Web Maps, Experiences) in a presentation-ready format

Next Steps

  • Each group member created an ArcGIS Hub website to embed the Experiences

Milestone 5: Final Submission

Outcomes

  • One ArcGIS Hub was selected to embed the Experiences along with explanatory text and web maps to support the problem statement

Future Improvements

  • More functionality and interactivity in the Experiences and in the ArcGIS Hub website
